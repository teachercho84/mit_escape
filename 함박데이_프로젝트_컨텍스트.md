# 함박데이 MIT 연구소 탈출 — 운영 홈페이지 개발 컨텍스트

## 1. 프로젝트 개요

- **행사명**: 함박데이 MIT 연구소 탈출 (ARIA 연구소 탈출 프로젝트)
- **주최**: 영산고등학교 공학·코딩 동아리 (MIT 동아리)
- **형식**: 방탈출 체험 부스
- **목적**: 아두이노 코딩, 회로 복구, RC카 자율주행을 체험하는 교육형 방탈출
- **운영 홈페이지 GitHub 레포**: `aria-escape-leaderboard`

---

## 2. 행사 운영 방식

| 항목 | 내용 |
|------|------|
| 운영 방식 | 릴레이식 (앞 팀 클리어 시 다음 팀 입장) |
| 팀당 인원 | 2~4명 (권장 3명) |
| 사전 교육 | 10분 (체험 전 아두이노·회로 기초 교육) |
| 체험 제한시간 | 15분 |
| 힌트 페널티 | 1회 사용 시 +30초 가산 |
| 등수 산정 기준 | 체험 시작부터 완료까지 소요시간 (힌트 페널티 포함) |
| 2시간 예상 참여 | 약 24~28팀 / 72~84명 |

---

## 3. 스테이지 구성

### Stage 1 — 모스부호 송신기 코딩
- 체험 학생이 아두이노 IDE에서 `dot()` / `dash()` 함수를 빈칸에 채워 업로드
- LED가 모스부호 신호를 출력하면 해독자가 모스부호 해독표로 글자를 맞힘
- 목표시간: 4~5분

### Stage 2 — 파손 회로 복구
- 체험 학생이 브레드보드와 아두이노만 보고 끊어진 배선 3곳에 점퍼선 연결
- 회로도는 힌트카드 #1로 숨겨져 있음
- 정답 시 초록 LED + 성공음, 오답 시 빨간 LED + 경보음
- 목표시간: 2~3분

### Stage 4 — RC카 자율주행 코딩
- 체험 학생이 `goForward()` / `turnRight()` / `turnLeft()` 함수를 loop() 빈칸에 채움
- 업로드 후 버튼을 누르면 RC카가 ㄷ자 코스를 자율주행
- 목표시간: 4~5분

## 4. 힌트카드 구성

| 스테이지 | 힌트카드 #1 | 힌트카드 #2 |
|----------|------------|------------|
| Stage 1 | 전송해야 할 글자 | dot()/dash() 함수 개념 |
| Stage 2 | 회로도 | 핀 번호 + 브레드보드 위치 |
| Stage 4 | 함수 목록 | ㄷ자 코스 평면도 |

## 5. 운영 홈페이지 요구사항

### 5-1. 화면 구성 (2가지)

#### A. 운영자 입력 화면 (관리자용)
진행요원이 팀 정보를 입력하는 화면.

필요한 입력 항목:
- 팀 이름 (텍스트)
- 소요시간 (분:초 형식 또는 초 단위)
- 힌트 사용 횟수 (0~6회, 스테이지당 최대 2회 × 3스테이지)
- 클리어 여부 (완주 / 미완주)

자동 계산:
- 최종 기록 = 소요시간 + (힌트 횟수 × 30초)

#### B. 리더보드 표시 화면 (참가자/관람용)
대형 화면 또는 노트북에 띄워놓는 화면.

표시 항목:
- 순위
- 팀 이름
- 소요시간 (순수 시간)
- 힌트 사용 횟수
- 최종 기록 (페널티 포함)
- 실시간 업데이트

### 5-2. 기능 요구사항

- 팀 등록 / 수정 / 삭제 가능
- 최종 기록 기준 자동 정렬 (오름차순)
- 미완주 팀은 완주 팀 아래에 별도 표시
- 1위~3위 강조 표시 (금/은/동 느낌)
- 모바일 대응 (운영자가 스마트폰으로 입력 가능)
- 별도 서버 없이 동작 가능하면 좋음 (로컬 또는 GitHub Pages)

### 5-3. 디자인 방향

- **테마**: MIT 연구소 / ARIA 시스템 느낌
- **색상**: 네이비(#1F3864) + 흰색 + 포인트 컬러 (금색 또는 파랑)
- **분위기**: 깔끔하고 공식적인 느낌. SF 연구소 모니터 화면 같은 느낌도 좋음
- **폰트**: 고딕 계열, 숫자는 모노스페이스

## 7. 참고 사항

- 행사 당일 인터넷 연결이 불안정할 수 있으므로 오프라인에서도 동작해야 함
- 운영자(동아리 학생)가 스마트폰으로 쉽게 입력할 수 있어야 함
- 리더보드는 별도 노트북/태블릿에 항상 띄워놓는 방식으로 운영
- 팀 수가 최대 28팀이므로 성능 이슈는 없음
- 행사 종료 후 결과를 캡처/저장할 수 있으면 좋음
