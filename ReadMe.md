# ARIA 연구소 탈출 — 리더보드 (aria-escape-leaderboard)

영산고등학교 MIT 동아리 **함박데이 방탈출 부스** 운영 홈페이지입니다.
서버 없이 브라우저만으로 동작합니다 (LocalStorage 저장, 완전 오프라인 가능).

## 실행 방법

`index.html` 파일을 브라우저(크롬 권장)로 열기만 하면 됩니다. 인터넷 연결 불필요.

> ⚠️ 기록은 **그 브라우저의 LocalStorage**에 저장됩니다. 행사 내내 **같은 노트북, 같은 브라우저**를 사용하세요. 시크릿 모드는 사용 금지(닫으면 기록이 사라짐).

## 화면 구성

| 파일 | 용도 |
|------|------|
| `index.html` | 메인 화면 — 팀 입장 / 진행 중 타이머 / 리더보드 |
| `admin.html` | 관리 콘솔 — 기록 수정·삭제·수동 추가·CSV 내보내기·전체 초기화 |

## 운영 방법 (행사 당일)

1. **입장**: 학생이 상단 입력칸에 팀명을 입력하고 **Enter** → 그 순간부터 타이머 시작
   - 여러 팀이 동시에 진행 가능 (1팀이 Stage 2로 넘어가면 2팀 입장 OK)
   - 같은 팀명은 중복 입장 불가
2. **힌트**: 진행요원이 해당 팀 카드의 **[힌트 +1]** 버튼 클릭 (1회당 최종기록 +30초, 최대 6회)
3. **종료**: 해당 팀 카드의 **[탈출 성공]** 또는 **[미완주]** 클릭 → 확인 팝업 후 리더보드에 자동 등록
   - 15분 초과 시 타이머가 빨간색으로 경고 (자동 종료는 안 됨 — 진행요원이 판단)
4. **실수 복구**: 우측 상단 ⚙ 아이콘 → 관리 콘솔에서 기록 수정/삭제/수동 추가
5. **행사 종료 후**: 관리 콘솔에서 **[CSV 내보내기]**로 결과 저장

### 새로고침해도 괜찮나요?

네. 진행 중인 타이머는 시작 시각 기준으로 계산되므로, 실수로 새로고침하거나 브라우저를 껐다 켜도 시간이 정확하게 이어집니다.

### 리허설 후 초기화

관리 콘솔의 **[전체 초기화]** 버튼 (이중 확인 후 모든 기록 삭제 — 본 행사 시작 전에만 사용).

## 점수 계산

```
최종 기록 = 소요시간 + (힌트 사용 횟수 × 30초)
```

- 완주 팀: 최종 기록 오름차순 정렬, 1~3위 금/은/동 강조
- 미완주 팀: 리더보드 하단 별도 표시

## 파일 구조

```
├── index.html          # 메인 화면 (입장/타이머/리더보드)
├── admin.html          # 관리 콘솔
├── css/style.css       # ARIA 연구소 테마
└── js/
    ├── storage.js      # LocalStorage 데이터 관리
    ├── main.js         # 타이머·리더보드 로직
    └── admin.js        # 관리 콘솔 로직
```

## 기술

순수 HTML + CSS + JavaScript. 빌드 도구·백엔드·외부 라이브러리 없음. GitHub Pages 배포 가능.
