## 1. 완료된 것 ✅

오프라인 스톱워치형 리더보드 **완성 + 검증 완료** (Playwright 자동 테스트 24개 전부 통과).

### 만든 파일

| 파일 | 역할 |
|------|------|
| `index.html` + `js/main.js` | 메인 화면 — 팀 입장 / 동시 타이머 / 리더보드 |
| `admin.html` + `js/admin.js` | 관리 콘솔 — 기록 수정·삭제·수동 추가·CSV 내보내기·전체 초기화 |
| `js/storage.js` | LocalStorage 데이터 모듈 (모든 저장/읽기가 이 파일에 모여 있음) |
| `css/style.css` | ARIA 연구소 테마 (네이비 #1F3864 + 금색, 스캔라인, 모노스페이스) |
| `ReadMe.md` | 동아리 학생용 운영 매뉴얼 |

### 핵심 동작 (원래 계획에서 변경된 부분 포함)

- **스톱워치 방식**: 운영자가 시간을 수동 입력하는 게 아니라, 학생이 팀명 입력 + Enter 하면 그 순간부터 타이머 시작
- **동시 여러 팀 진행 가능**: 파이프라인식 운영 지원 (1팀이 Stage 2로 가면 2팀이 Stage 1 입장).
  팀마다 독립 타이머 카드 + 각자 [힌트 +1] [탈출 성공] [미완주] 버튼
- 최종 기록 = 소요시간 + 힌트 × 30초 (자동 계산), 최종기록 오름차순 정렬, 1~3위 금/은/동
- 미완주 팀은 리더보드 하단 별도 표시
- 새로고침/브라우저 재시작해도 타이머 이어짐 (시작 시각 기준 계산)
- 완전 오프라인 동작 (LocalStorage), 중복 팀명 차단, 15분 초과 시 빨간 경고

### 버그 수정 이력

- 리더보드 숫자 열(소요시간/힌트/최종기록) 헤더-값 정렬 어긋남 → CSS 우선순위 문제 수정 완료
  (`css/style.css`의 `.board th.col-num` 규칙)

## 2. 진행 중 / 미결정 사항

### git 커밋이 아직 없음 ⚠️

레포에 커밋이 0개 상태. 다음 세션 시작 시 먼저 커밋 권장:
```
git add . ; git commit -m "initial: 오프라인 리더보드 완성"
```

### 다음 작업 후보: 온라인 실시간 관람 기능 (논의 완료, 구현 전)

요구사항이 정리된 상태:
- **기록 입력은 행사장 PC 1대에서만** (지금 방식 그대로, LocalStorage가 원본)
- **그 기록을 외부에서 온라인으로 구경**할 수 있게 하고 싶음 (학부모/대기 학생이 폰으로)

합의된 설계 (단방향 동기화 — 한 대가 쓰고 여러 명이 읽는 구조라 충돌 없음):

```
행사장 PC (지금 그대로)                    온라인 관람자들
┌─────────────────────┐                ┌──────────────────┐
│ index.html / admin   │   기록 변경 시   │ view.html (신규)  │
│ LocalStorage (원본)  │ ──→ Firebase ──→ │ 읽기 전용 리더보드  │
└─────────────────────┘    실시간 DB     └──────────────────┘
```

- 행사장 PC는 인터넷이 끊겨도 행사 진행에 지장 없음 (전송만 잠시 실패, 복구 시 재동기화)
- 진행 중인 팀의 startEpochMs까지 동기화하면 온라인에서도 실시간 타이머 관람 가능
- 보안: 하루짜리 행사이므로 간단한 쓰기 키(행사장 PC만 아는 값)로 충분

구현 시 필요한 것:
1. Firebase 프로젝트 생성 (선생님 구글 계정, 무료, ~5분) — 단계별 안내 예정
2. 동기화 모듈 추가 (`storage.js`에 저장이 모여 있어서 변경 범위 작음)
3. `view.html` 읽기 전용 페이지 신규 작성
4. GitHub Pages 배포 (view.html 접속 주소용)
